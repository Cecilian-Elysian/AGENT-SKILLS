# Obsidian Canvas 创建器

将 wiki 文件内容转换为 Obsidian .canvas 格式的可视化画布。

## 工作流程

### 1. 读取源文件

读取用户提供的本地 wiki 文件路径。分析文件内容，提取：
- 主要概念/主题
- 关键节点/要点
- 概念之间的关系
- 重要引用或链接

### 2. 设计 Canvas 布局

根据内容选择布局策略：

**预设模板：**
- **中心辐射型**：核心概念在中心，相关概念围绕展开
- **流程型**：按逻辑顺序线性排列（上下或左右）
- **分组型**：相关内容聚类分组
- **自由型**：根据概念重要性自动定位

**节点定位规则：**
- 核心主题放在中心
- 相关节点按距离中心的位置分层
- 节点之间保持适当间距（最小 200px）
- 优先使用垂直/水平对齐

### 3. 生成 Canvas JSON

输出标准 Obsidian Canvas JSON 格式。

## 输出格式

始终输出符合以下结构的 `.canvas` 文件：

```json
{
  "nodes": [
    {
      "id": "唯一标识符",
      "type": "text",
      "text": "卡片文本内容",
      "width": 宽度,
      "height": 高度,
      "x": X坐标,
      "y": Y坐标
    },
    {
      "id": "图片节点",
      "type": "image",
      "file": "图片路径",
      "width": 宽度,
      "height": 高度,
      "x": X坐标,
      "y": Y坐标
    },
    {
      "id": "链接节点",
      "type": "link",
      "link": "链接目标",
      "title": "显示标题",
      "width": 宽度,
      "height": 高度,
      "x": X坐标,
      "y": Y坐标
    }
  ],
  "edges": [
    {
      "id": "边标识符",
      "fromNode": "源节点id",
      "fromSide": "right",
      "toNode": "目标节点id",
      "toSide": "left"
    }
  ]
}
```

## 节点类型说明

| 类型 | 用途 | 必需字段 |
|------|------|----------|
| text | 纯文本内容卡片 | text |
| image | 嵌入图片 | file (路径) |
| link | 引用其他笔记 | link (目标路径) |

## 布局参数

**画布设置：**
- 默认画布大小：适合内容自动扩展
- 节点最小宽度：200px
- 节点最大宽度：400px
- 节点间最小间距：200px

**坐标系统：**
- 原点 (0, 0) 在画布左上角
- X 轴向右增加
- Y 轴向下增加

## 内容处理指南

1. **文本卡片**：提取关键句子，控制在 100-200 字以内
2. **链接节点**：识别文件引用、wiki 链接、外部链接
3. **关系识别**：根据标题层级、列表结构、引用关系建立边连接
4. **图片处理**：保留图片路径引用，图片节点独立放置

## 输出文件

- 扩展名：`.canvas`
- 格式：JSON
- 保存位置：与源文件相同目录，或用户指定位置
- 文件名建议：`源文件名.canvas` 或 `canvas-日期.canvas`

## 示例

**输入：** 用户提供 `notes/wiki-concept.md`，内容关于"设计模式"

**输出：** 生成 `notes/wiki-concept.canvas`，包含：
- 中心节点："设计模式"（text 类型）
- 子类别节点：创建型、结构型、行为型（text 类型）
- 具体模式节点：工厂方法、单例、观察者等（text 类型）
- 边连接相关概念

## 错误处理

- 文件不存在：提示用户检查路径
- 内容为空：提示用户提供有效内容
- 格式错误：确保输出符合 Canvas JSON 规范

## 注意事项

- 节点 ID 必须唯一，使用简短描述性字符串
- 所有坐标和尺寸应为数字（px 单位内联）
- 边连接必须引用有效的节点 ID
- text 类型的 text 字段支持 Markdown 格式
