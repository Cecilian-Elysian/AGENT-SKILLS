算法哲学是一种通过代码表达的计算机美学运动。输出 .md 文件（哲学理念）、.html 文件（交互式查看器）和 .js 文件（生成算法）。

1. 算法哲学创作（.md 文件）
2. 通过创建 p5.js 生成艺术来表达（.html + .js 文件）

首先，完成以下任务：

## 算法哲学创作

首先，创建一个算法哲学（不是静态图像或模板），它将通过以下方式被诠释：
- 计算过程、涌现行为、数学之美
- 随机种子、噪声场、有机系统
- 粒子、流动、场、力
- 参数变化与可控混沌

### 关键理解
- 接收到的：用户提供的微妙输入或指示，作为基础但不限制创作自由
- 创作出的：算法哲学/生成美学运动
- 接下来会发生什么：同一版本接收哲学理念并通过代码表达——创建 90% 算法生成、10% 必要参数的 p5.js 草图

考虑这种方法：
- 为一个生成艺术运动撰写宣言
- 下一阶段是编写将理念变为现实的算法

哲学必须强调：算法表达。涌现行为。计算之美。随机变化。

### 如何生成算法哲学

**命名这个运动**（1-2个词）："有机湍流" / "量子和谐" / "涌现静谧"

**阐述哲学理念**（4-6段——简洁但完整）：

为捕捉算法本质，表达这个哲学如何通过以下方式呈现：
- 计算过程和数学关系？
- 噪声函数和随机模式？
- 粒子行为和场动力学？
- 时间演化和系统状态？
- 参数变化和涌现复杂性？

**关键指南：**
- **避免冗余**：每个算法方面只提及一次。避免重复关于噪声理论、粒子动力学或数学原理的概念，除非添加新的深度。
- **反复强调工艺**：哲学必须多次强调最终算法看起来像是花费了无数小时开发的，经过精心打磨，来自绝对顶尖领域的人。这些框架是必不可少的——重复使用如"精心雕琢的算法"、"深厚计算专业知识的产物"、"精心优化"、"大师级实现"等短语。
- **保留创意空间**：在算法方向上要具体，但足够简洁，以便下一个 Claude 能够在极高的工艺水平上进行诠释性实现选择。

哲学必须引导下一个版本以算法方式表达理念，而不是通过静态图像。美存在于过程中，而不是最终帧中。

### 哲学示例

**"有机湍流"**
哲学：被自然法则约束的混沌，从无序中涌现秩序。
算法表达：由分层 Perlin 噪声驱动的流场。数以千计的粒子沿着向量力移动，它们的轨迹累积成有机密度图。多重噪声倍频创建湍流区域和静谧区域。颜色从速度和密度中涌现——快速粒子燃烧明亮，慢速粒子消退为阴影。算法运行直到平衡——精心调谐的平衡，每个参数都经过大师级计算美学专家无数次迭代打磨。

**"量子和谐"**
哲学：表现出波状干涉图案的离散实体。
算法表达：在网格上初始化的粒子，每个粒子携带通过正弦波演化的相位值。当粒子靠近时，它们的相位干涉——相长干涉创建明亮节点，相消干涉创建空洞。简单的谐波运动产生复杂的涌现曼陀罗。经过精心频率校准的产物，每个比率都经过精心选择以产生共振之美。

**"递归低语"**
哲学：跨尺度的自相似性，有限空间中的无限深度。
算法表达：递归细分的分支结构。每个分支略微随机化但受黄金比例约束。L-系统或递归细分生成感觉既数学又有机树状形式。微妙的噪声扰动打破完美对称性。线宽随每个递归级别递减。每个分支角度都是深度数学探索的产物。

**"场动力学"**
哲学：不可见的力量通过其对物质的影响变得可见。
算法表达：由数学函数或噪声构建的向量场。粒子在边缘诞生，沿着场线流动，当达到平衡或边界时消亡。多个场可以吸引、排斥或旋转粒子。可视化只显示轨迹——幽灵般的不可见力量证据。通过力量平衡精心编排的计算舞蹈。

**"随机结晶"**
哲学：随机过程结晶成有序结构。
算法表达：随机圆填充或 Voronoi 镶嵌。从随机点开始，让它们通过松弛算法演化。单元格推开直到平衡。根据单元格大小、邻居数量或距中心距离着色。最终的有机镶嵌感觉既随机又必然。每个种子产生独特的结晶之美——大师级生成算法的标志。

*这些是精简示例。实际的算法哲学应该是 4-6 段实质性内容。*

### 基本原则
- **算法哲学**：创建通过代码表达的计算机世界观
- **过程优于产品**：始终强调美从算法的执行中涌现——每次运行都是独特的
- **参数表达**：理念通过数学关系、力量、行为来传达——不是静态构图
- **艺术自由**：下一个 Claude 以算法方式诠释哲学——提供创意实现空间
- **纯粹生成艺术**：这是关于创造活的算法，不是带有随机性的静态图像
- **专家工艺**：反复强调最终算法必须感觉是精心雕琢的，经过无数次迭代打磨，是计算美学领域绝对顶尖专家的深度专业知识产物

**算法哲学应该有 4-6 段长。** 用诗意的计算哲学填充它，汇聚预期愿景。避免重复相同的观点。将此算法哲学作为 .md 文件输出。

### 技术要求

**随机种子（Art Blocks 模式）**：
```javascript
// 始终使用种子以确保可重现性
let seed = 12345; // 或从用户输入哈希
randomSeed(seed);
noiseSeed(seed);
```

**参数结构 - 遵循哲学**：

为从算法哲学自然涌现的参数建立参数，考虑："这个系统的哪些属性可以调整？"

```javascript
let params = {
  seed: 12345,  // 始终包含种子以确保可重现性
  // 颜色
  // 添加控制算法的参数：
  // - 数量（多少？）
  // - 比例（多大？多快？）
  // - 概率（多可能？）
  // - 比例（什么比例？）
  // - 角度（什么方向？）
  // - 阈值（行为何时改变？）
};
```

**要设计有效的参数，关注系统需要可调谐的属性，而不是考虑"图案类型"。**

**核心算法 - 表达哲学**：

**关键**：算法哲学应该决定构建什么。

通过代码表达哲学时，避免思考"我应该使用哪种图案？"而是思考"如何通过代码表达这个哲学？"

如果哲学是关于**有机涌现**，考虑使用：
- 随着时间积累或增长的元素
- 被自然规则约束的随机过程
- 反馈循环和交互

如果哲学是关于**数学之美**，考虑使用：
- 几何关系和比例
- 三角函数和谐波
- 创造意想不到图案的精确计算

如果哲学是关于**可控混沌**，考虑使用：
- 严格边界内的随机变化
- 分岔和相变
- 从无序中涌现秩序

**算法从哲学流动，而不是从选项菜单流动。**

在指导下，让概念本质启发创意和原创选择。构建表达这个特定请求愿景的东西。

**画布设置**：标准 p5.js 结构：
```javascript
function setup() {
  createCanvas(1200, 1200);
  // 初始化系统
}

function draw() {
  // 生成算法
  // 可以是静态的（noLoop）或动画的
}
```

### 工艺要求

**关键**：要达到大师级水平，创建感觉像是由大师级生成艺术家通过无数次迭代涌现的算法。精心调谐每个参数。确保每个图案都有目的地涌现。这不是随机噪声——这是通过深度专业知识打磨的可控混沌。

- **平衡**：复杂性而无视觉噪声，秩序而无僵化
- **色彩和谐**：深思熟虑的调色板，而不是随机 RGB 值
- **构图**：即使在随机中，也保持视觉层次和流动
- **性能**：流畅执行，如果动画则优化为实时
- **可重现性**：相同的种子总是产生相同的输出

输出：

## 交互式工件创建

**提醒：`templates/viewer.html` 应该已经阅读（见步骤 0）。使用该文件作为起点。**

为了允许探索生成艺术，创建一个单一的、自包含的 HTML 工件。确保此工件可立即在 claude.ai 或任何浏览器中工作——无需设置。所有内容内联嵌入。

### 关键：什么是固定的 vs 可变的

`templates/viewer.html` 文件是基础。它包含所需的确切结构和样式。

**固定（始终按原样包含）：**
- 布局结构（页眉、侧边栏、主画布区域）
- Anthropic 品牌（UI 颜色、字体、渐变）
- 侧边栏中的种子部分：
  - 种子显示
  - 上一个/下一个按钮
  - 随机按钮
  - 跳转种子输入 + 跳转按钮
- 侧边栏中的操作部分：
  - 重新生成按钮
  - 重置按钮

**可变（为每个艺术品自定义）：**
- 整个 p5.js 算法（setup/draw/类）
- 参数对象（定义艺术品需要什么）
- 侧边栏中的参数部分：
  - 参数控件数量
  - 参数名称
  - 滑块的最小/最大/步长值
  - 控件类型（滑块、输入等）
- 颜色部分（可选）：
  - 某些艺术品需要颜色选择器
  - 某些艺术品可能使用固定颜色
  - 某些艺术品可能是单色的（不需要颜色控件）
  - 根据艺术品需要决定

**每个艺术品应该有独特的参数和算法！** 固定部分提供一致的 UX——其他一切表达独特愿景。

### 必需功能

**1. 参数控件**
- 数值参数的滑块（粒子数量、噪声比例、速度等）
- 调色板颜色的颜色选择器
- 参数变化时实时更新
- 重置按钮恢复默认值

**2. 种子导航**
- 显示当前种子编号
- "上一个"和"下一个"按钮循环种子
- "随机"按钮获取随机种子
- 输入字段跳转到特定种子
- 按需生成 100 个变体（种子 1-100）

**3. 单个工件结构**
```html
<!DOCTYPE html>
<html>
<head>
  <!-- 来自 CDN 的 p5.js - 始终可用 -->
  <script src="https://cdnjs.cloudflare.com/ajax/libs/p5.js/1.7.0/p5.min.js"></script>
  <style>
    /* 所有样式内联 - 简洁、最少 */
    /* 画布在上，控件在下 -->
  </style>
</head>
<body>
  <div id="canvas-container"></div>
  <div id="controls">
    <!-- 所有参数控件 -->
  </div>
  <script>
    // 所有 p5.js 代码内联在这里
    // 参数对象、类、函数
    // setup() 和 draw()
    // UI 处理程序
    // 所有内容自包含
  </script>
</body>
</html>
```

**关键**：这是一个单一工件。没有外部文件，没有导入（除了 p5.js CDN）。所有内容内联。

**4. 实现细节 - 构建侧边栏**

侧边栏结构：

**1. 种子（固定）** - 始终按原样包含：
- 种子显示
- 上一个/下一个/随机/跳转按钮

**2. 参数（可变）** - 为艺术品创建控件：
```html
<div class="control-group">
    <label>参数名称</label>
    <input type="range" id="param" min="..." max="..." step="..." value="..." oninput="updateParam('param', this.value)">
    <span class="value-display" id="param-value">...</span>
</div>
```
添加与参数数量相同的 control-group div。

**3. 颜色（可选/可变）** - 如果艺术品需要可调颜色则包含：
- 如果用户应该控制调色板则添加颜色选择器
- 如果艺术品使用固定颜色则跳过此部分
- 如果艺术品是单色的则跳过（不需要颜色控件）

**4. 操作（固定）** - 始终按原样包含：
- 重新生成按钮
- 重置按钮
- 下载 PNG 按钮

**要求**：
- 种子控件必须工作（上/下/随机/跳转/显示）
- 所有参数必须有 UI 控件
- 重新生成、重置、下载按钮必须工作
- 保持 Anthropic 品牌（UI 样式，不是艺术颜色）

### 使用工件

HTML 工件可立即工作：
1. **在 claude.ai 中**：显示为交互式工件——立即运行
2. **作为文件**：保存并在任何浏览器中打开——无需服务器
3. **分享**：发送 HTML 文件——它完全自包含

## 变体与探索

工件默认包含种子导航（上/下/随机按钮），允许用户在不创建多个文件的情况下探索变体。如果用户想要突出显示特定变体：

- 包含种子预设（"变体 1：种子 42"、"变体 2：种子 127"等按钮）
- 添加"画廊模式"并排显示多个种子的缩略图
- 所有内容在同一个单一工件中

这就像从同一块印版创建一系列印刷品——算法是一致的，但每个种子揭示其潜力的不同方面。交互性意味着用户通过探索种子空间发现自己的最爱。

## 创意过程

**用户请求** → **算法哲学** → **实现**

每个请求都是独特的。过程包括：

1. **解释用户意图** - 正在寻找什么美学？
2. **创建算法哲学**（4-6段）描述计算方法
3. **用代码实现** - 构建表达这个哲学的算法
4. **设计适当的参数** - 什么应该是可调的？
5. **构建匹配的 UI 控件** - 那些参数的滑块/输入

**常量**：
- Anthropic 品牌（颜色、字体、布局）
- 种子导航（始终存在）
- 自包含 HTML 工件

**其他都是可变的**：
- 算法本身
- 参数
- UI 控件
- 视觉结果

为获得最佳结果，相信创意，让哲学指导实现。
